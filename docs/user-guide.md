# 用户指南

本指南详细介绍 Uaha 的各项功能和使用方法。

## 界面概览

![Uaha 界面演示](../assets/uaha.gif)

## 模型管理

### 浏览可用模型

模型列表页面显示所有可用的 AI 模型，包括：

- **模型名称** - 模型的显示名称
- **模型大小** - 下载后占用的磁盘空间
- **状态** - 模型当前状态
  - `available` - 可下载
  - `downloading` - 下载中
  - `installed` - 已安装
  - `error` - 下载或安装错误
- **进度** - 下载进度百分比（0-100%）

### 下载模型

1. 在模型列表中找到需要的模型
2. 点击「下载」按钮
3. 等待下载完成（进度条会实时显示）
4. 下载完成后状态变为「已安装」

### 删除模型

1. 在模型列表中找到已安装的模型
2. 点击「删除」按钮
3. 确认删除操作

**注意**: 删除模型后，依赖该模型的服务将无法启动。

## API 服务管理

### 创建服务实例

1. 进入「API 服务」页面
2. 点击「创建服务」按钮
3. 配置服务参数：
   - **服务名称** - 服务的显示名称
   - **模型** - 选择已安装的模型
   - **端口** - 服务监听端口（默认自动分配）
4. 点击「创建」

### 启动服务

1. 在服务列表中找到已创建的服务
2. 点击「启动」按钮
3. 等待服务状态变为「运行中」

### 停止服务

1. 在服务列表中找到运行中的服务
2. 点击「停止」按钮
3. 确认停止操作

### 服务状态说明

| 状态 | 描述 |
|------|------|
| `stopped` | 服务已停止 |
| `running` | 服务运行中 |
| `error` | 服务启动或运行错误 |

### 服务监控

运行中的服务会显示以下信息：

- **进程 ID (PID)** - 服务进程标识
- **内存使用** - 当前内存占用
- **GPU 内存** - GPU 显存占用（如适用）
- **延迟** - 请求响应时间（毫秒）

## 配置管理

### Aha 引擎配置

- **Aha 可执行文件路径** - Aha 引擎的完整路径
  - macOS: `/usr/local/bin/aha`
  - Windows: `C:\Program Files\Aha\aha.exe`
  - Linux: `/usr/bin/aha`

### 模型下载设置

- **模型下载目录** - 模型文件存储位置
- **下载重试次数** - 下载失败后的重试次数（默认：3）

### 主题和外观

- **主题** - 选择「亮色」或「深色」主题
- **主色调** - 自定义应用主色调（RGB 值）
- **圆角大小** - UI 元素的圆角半径
- **字体大小** - 全局字体大小（像素）

### 其他设置

- **自动启动** - 开机时自动启动应用
- **自动更新** - 自动检查并下载更新
- **语言** - 界面语言（当前仅支持中文）

## 常见使用场景

### 场景一：快速部署本地 LLM 服务

1. 下载所需的 LLM 模型（如 Llama 2）
2. 创建一个新的 API 服务
3. 选择已下载的模型
4. 启动服务
5. 使用默认端口访问 API（如 `http://localhost:11011`）

### 场景二：多模型并行服务

1. 下载多个不同模型（如一个用于对话，一个用于代码生成）
2. 为每个模型创建独立的服务实例
3. 使用不同端口避免冲突
4. 同时运行多个服务

### 场景三：切换深色模式

1. 进入「配置」页面
2. 在「主题」下拉菜单中选择「深色」
3. 点击「保存」
4. 界面立即切换到深色主题

### 场景四：管理磁盘空间

1. 进入「模型列表」页面
2. 查看每个模型的大小
3. 删除不再使用的模型释放空间
